# 매크로 브리핑 대시보드

## Problem Statement

HMW — 매일 미국채 금리와 원달러 환율을 확인하는 사람이, 숫자의 움직임이 아니라 "왜 움직였는가"를 뉴스 맥락과 함께 한국어로 즉시 파악하게 할 수 있을까?

## Recommended Direction

매일 정해진 시간에 자동 갱신되는 **큐레이션 브리핑 웹 대시보드**. 미국채 10년물 금리와 원달러 환율의 주간 추이 차트를 보여주고, 각 지표에 연결된 주목받은 기사를 한국어 요약과 함께 제공한다.

핵심 설계 결정은 **지표별 언어 분리**다. 미국채 금리는 글로벌 요인이 지배적이므로 영문 신뢰 언론사(Bloomberg·FT·WSJ·Reuters) 기사를 우선하고, 원달러 환율은 한국 정책·수급 맥락이 중요하므로 한글 신뢰 언론사(한국경제·매일경제·연합뉴스·조선비즈)를 우선한다. 뉴스 선별 기준은 공감·좋아요 수 → 언론사 지명도 티어 → 최신성 순으로 적용한다.

대상은 매크로 지표를 매일 확인하는 개인 투자자, 직장인, 학생. 영어 독해 없이도 글로벌 금융 맥락을 소화할 수 있게 하는 것이 차별점이다.

## Key Assumptions to Validate

- [ ] 네이버 뉴스 공감수를 API 또는 RSS로 가져올 수 있다 — NewsAPI·네이버 뉴스 검색 API 프로토타입으로 확인
- [ ] Bloomberg·FT·WSJ RSS에서 "미국채 10년물 금리" 관련 기사를 키워드 필터로 정확하게 추릴 수 있다 — 샘플 50건 수작업 검증
- [ ] Claude API 번역·요약 결과가 파파고·구글 번역보다 맥락을 잘 담는다고 사용자가 느낀다 — 5인 소규모 사용자 테스트

## MVP Scope

- 지표 2개: 미국채 10년물 금리, 원달러 환율
- 각 지표별 주간 추이 차트 1개
- 각 지표별 기사 1-2건 (출처 링크 + 한국어 번역 + 3줄 요약)
- 웹 페이지 1장, 공유 링크 제공
- 매일 정해진 시간 자동 갱신

## Not Doing (and Why)

- 알림·구독 기능 — 핵심 가정(뉴스 발굴 품질) 검증 전에 배포 인프라 투자 불필요
- 코스피·유가 등 추가 지표 — 지표 2개로도 "지표+뉴스+번역" 루프가 작동하는지 먼저 확인
- 사용자 맞춤 설정·로그인 — 퍼스널라이제이션은 core value 증명 후 2단계

## Open Questions

- 네이버 뉴스 공감수 API 또는 크롤링 허용 여부 확인 필요
- 언론사 지명도 티어를 누가, 어떤 기준으로 관리할 것인가 (초기에는 하드코딩)
- 자동 갱신 주기: 하루 1회(오전 7시)로 충분한가, 장중 갱신이 필요한가
